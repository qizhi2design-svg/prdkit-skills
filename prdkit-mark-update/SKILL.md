# prdkit-mark-update

修改原型页面上的标注内容，适应需求变化。

## 使用场景

- 需求调整了，标注说明需要更新
- 发现标注写得不够清楚，需要补充
- 标注内容有错误，需要修正

## 执行流程

### 第 1 步：确定原型路径并展示所有标注

首先需要知道要修改哪个原型的标注：

1. **询问原型路径**（如果产品经理没有明确说明）：
   - "您想修改哪个原型的标注？"
   - 原型路径示例：`需求管理后台`、`dashboard`、`用户中心/个人设置`

2. **列出所有标注**，使用命令：
   ```bash
   prdkit mark list --prototype {原型路径}
   ```

3. **清晰展示**当前所有标注：
   ```
   📍 标注 #1：顶部概览区
   ID: mark-1777342000001
   位置: body > div.app-shell > div.content-shell > header.topbar
   
   📍 标注 #2：登录按钮
   ID: mark-1777342000002
   位置: body > div.login-form > button.submit
   
   📍 标注 #3：忘记密码链接
   ID: mark-1777342000003
   位置: body > div.login-form > a.forgot-password
   ```

4. 询问产品经理：**"您想修改哪个标注？"**（可以说标注标题或编号）

### 第 2 步：获取标注详情并分析修改原因

在产品经理选择要修改的标注后：

1. **获取标注的完整内容**，使用命令：
   ```bash
   prdkit mark get {mark-id} --prototype {原型路径}
   ```

2. **展示这个标注的详细内容**：
   ```
   📍 标注详情
   
   标题：顶部概览区
   ID：mark-1777342000001
   位置：body > div.app-shell > div.content-shell > header.topbar
   
   当前说明：
   后台首页第一屏建议先回答运营或管理角色最关心的两个问题：
   
   - 现在整体状态怎么样
   - 我接下来最该点哪里
   
   标题、副标题和右侧主按钮要形成清晰的优先级。
   ```

3. **询问修改原因**（用简单的语言）：
   - "为什么要修改这个标注？"
   - 提供常见原因选项：
     - 需求调整了（业务规则变了、功能范围变了）
     - 说明不够清楚（需要补充细节、需要更明确的描述）
     - 发现错误（理解有误、信息过时）
     - 其他原因

4. **根据修改原因提供建议**：
   - 如果是"需求调整"：提醒产品经理检查是否影响其他相关标注
   - 如果是"说明不够清楚"：建议补充哪些方面（功能说明、交互细节、数据规则）
   - 如果是"发现错误"：建议如何修正

### 第 3 步：收集新的标注内容

用简单的方式询问产品经理：

1. **"您想怎么改？"**
   - 可以直接说新的内容
   - 也可以说"在原来基础上补充..."
   - 可以只改标题，或只改说明，或两个都改

2. **智能理解修改意图**：
   - 如果产品经理说"补充..."，则在原内容基础上追加
   - 如果产品经理说"改成..."，则完全替换
   - 如果产品经理说"删除..."，则移除指定部分
   - 如果产品经理只说了新内容，询问是"替换"还是"补充"

3. **确认修改内容**：
   - 展示修改前后的对比
   - 询问："这样修改可以吗？"

### 第 4 步：更新标注

确认后，使用 CLI 命令更新标注：

**如果只修改标题：**
```bash
prdkit mark edit {mark-id} --prototype {原型路径} --title "新的标题"
```

**如果只修改说明内容（直接提供文本）：**
```bash
prdkit mark edit {mark-id} --prototype {原型路径} --desc "新的说明内容"
```

**如果修改说明内容（内容较长或有复杂格式）：**
1. 先将新内容写入临时文件（例如 `/tmp/mark-desc.md`）
2. 然后使用命令：
   ```bash
   prdkit mark edit {mark-id} --prototype {原型路径} --desc-file /tmp/mark-desc.md
   ```

**如果同时修改标题和说明：**
```bash
prdkit mark edit {mark-id} --prototype {原型路径} \
  --title "新的标题" \
  --desc-file /tmp/mark-desc.md
```

### 第 5 步：完成并引导下一步

修改完成后，告诉产品经理：

1. ✅ 标注已更新
2. 💡 建议下一步操作：
   - 如果需要修改其他相关标注：继续使用本 skill
   - 如果需要预览效果：使用 `prdkit serve` 命令启动预览服务器
   - 如果需要同步更新 PRD：使用 `prdkit prototype sync` 命令

## 注意事项

- **修改前一定要先展示当前内容**，避免产品经理不知道现在是什么样
- **用简单的语言询问**，不要问"变更类型"、"影响范围"这些抽象概念
- **提供修改建议**，而不是让产品经理从零开始想
- **标注文件位置**：`workspace/prototypes/{原型路径}/marks/mark-{timestamp}.md`
- **标注 ID 格式**：`mark-{timestamp}`，例如 `mark-1777342000001`
- **说明内容支持 Markdown 格式**：可以使用列表、加粗、链接等

## 产品经理友好的语言

- ✅ 说"标注说明"，不说"mark content"
- ✅ 说"需求调整"，不说"需求变更"
- ✅ 说"补充细节"，不说"增量更新"
- ✅ 说"原型路径"，不说"prototype path"
- ✅ 说"标注 ID"，不说"mark identifier"

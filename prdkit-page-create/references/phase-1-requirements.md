# 第一阶段：需求澄清

## 需求来源类型

### 文字描述场景
用户通过文字描述页面需求，需要通过提问澄清细节。

### 有参考截图/页面场景

**第一步：判断需求类型**

必须先确认用户的需求是"完全复刻"还是"参考设计"：

- **完全复刻**：布局、文案、样式都要保持一致（像素级还原）
- **参考设计**：只参考样式和布局，内容可以调整（灵感参考）

**第二步：针对性提问**

#### 完全复刻场景
用户要求保持布局、文案、样式完全一致，需要：
1. **让用户描述页面结构**：按从上到下的顺序列出所有元素（因为 AI 无法直接"看到"截图）
2. **提取视觉参数**：颜色、间距、圆角、字体大小、按钮样式
3. **生成 ASCII 线框图**：将用户描述转换为结构化的 wireframe
4. **确认还原细节**：哪些元素必须完全一致，哪些可以微调（如间距、颜色深浅）

#### 参考设计场景
用户只参考样式和布局，内容可调整，需要：
1. **分析参考内容**：识别布局结构、UI 元素、交互组件
2. **生成 ASCII 线框图**：将参考转换为结构化的 wireframe
3. **确认理解准确性**：让用户确认提取的结构是否正确
4. **补充业务细节**：参考无法体现的交互逻辑、数据层级、状态变化需要提问确认

## 提问原则

总共提 3-4 个问题即可，尽量混合用户视角、设计师视角和业务视角。可以按任务重写措辞，但问题必须具体、可回答。

## 第一优先确认

- 这是移动端、普通 Web 页面，还是 PC 后台/工作台？

## 用户视角问题

- 用户进入这个页面后，第一件必须完成的事是什么？
- 这个页面最重要的信息是"状态总览"、 "列表处理"，还是"单次操作提交"？
- 用户最常见的下一步动作是什么？返回、筛选、提交、查看详情，还是切换 tab？
- 哪些内容必须首屏可见，哪些内容可以折叠或下沉到二级操作？

## 功能模块视角问题

- 这个页面属于哪个功能模块？是独立功能还是某个流程的一部分？
- 页面需要与哪些其他模块或页面联动？（例如：从列表跳转到详情、提交后跳转到结果页）
- 是否需要权限控制？不同角色看到的内容或操作是否有差异？
- 数据来源是什么？是实时查询、定时刷新，还是一次性加载？
- 是否需要支持离线或弱网场景？

## 业务视角问题

- 这个页面解决什么业务问题？核心业务目标是什么？
- 用户在什么业务场景下会使用这个页面？
- 页面的关键业务指标是什么？（例如：转化率、完成率、处理效率）
- 是否有业务规则或约束需要在页面上体现？（例如：审批流程、状态流转规则）
- 异常情况如何处理？（例如：数据为空、操作失败、权限不足）

## 设计师视角问题

- 页面更偏"效率型工具"还是"品牌感/引导型界面"？
- 信息密度希望偏高还是偏轻？是否接受表格、分栏、侧边过滤这类高密度结构？
- 是否有现成品牌色、字体风格、圆角/阴影偏好，或者需要先按通用体系起稿？
- 是否需要重点设计空状态、异常状态、权限受限状态或加载过程？

## 完全复刻场景的特殊问题

当用户要求"完全保持一致"时，重点确认：

**第一个问题（必问）**：
- 这个页面需要完全复刻（布局、文案、样式都保持一致），还是只参考设计风格（内容可以调整）？

**结构确认问题**：
- 请按从上到下的顺序，详细描述页面包含哪些元素？
  - 例如：顶部 Logo（居中/居左）、标题文案（具体内容）、输入框（几个、标签文字）、按钮（文案、位置）、底部链接等
  - 如果有背景图或装饰元素，也请说明

**视觉参数问题**：
- 页面的视觉风格是什么样的？
  - 主色调（具体颜色，如 #1890ff 蓝色）
  - 按钮样式（圆角大小、实心/描边、阴影效果）
  - 间距风格（紧凑/宽松）
  - 字体大小（标题、正文、辅助文字）

**还原边界问题**：
- 哪些细节必须完全一致，哪些可以微调？
  - 文案内容（一字不差 / 可以调整措辞）
  - 颜色（精确匹配 / 相近即可）
  - 间距（像素级 / 视觉相似）
  - 图片/图标（使用原图 / 用占位符代替）

## 参考设计场景的特殊问题

当用户只参考设计风格时，重点确认：

- **结构确认**：我从参考中识别出的布局结构是否准确？（展示 ASCII 线框图）
- **交互逻辑**：点击/滑动/长按等交互行为的具体效果是什么？
- **数据层级**：列表数据的嵌套关系、筛选条件、排序规则是什么？
- **状态变化**：按钮点击后、表单提交后、数据加载中的状态如何展示？
- **边界情况**：数据为空、加载失败、权限不足时如何处理？

## 截图复刻场景的特殊问题

当用户提供截图要求复刻时，重点确认：

- **结构确认**：我从截图中识别出的布局结构是否准确？（展示 ASCII 线框图）
- **交互逻辑**：点击/滑动/长按等交互行为的具体效果是什么？
- **数据层级**：列表数据的嵌套关系、筛选条件、排序规则是什么？
- **状态变化**：按钮点击后、表单提交后、数据加载中的状态如何展示？
- **边界情况**：数据为空、加载失败、权限不足时如何处理？

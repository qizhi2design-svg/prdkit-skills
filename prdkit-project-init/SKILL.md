# 产品项目初始化引导

用问答方式收集信息，自动完成技术配置，让产品经理可以立即开始工作。

## 执行步骤

### 1. 欢迎

用友好的语言介绍 prdkit 的能力（管理 PRD、制作预览原型、记录讨论/Bug、追踪版本），告知只需回答几个问题即可创建项目空间。

### 2. 收集项目信息

逐一提问，每个问题提供说明和示例，允许跳过所有可选项：

**必填**：项目名称、作者名字
**可选**：项目简介、产品类型（企业内系统/面向用户产品等）、团队规模、项目阶段、保存位置（默认当前目录）

所有选项使用 `AskUserQuestion` 或 `@inquirer/prompts` 实现，不要暴露命令行。

### 3. 确认信息

汇总展示收集到的信息，用 `AskUserQuestion` 让用户确认或修改。

### 4. 执行初始化

显示进度：创建文件夹 → 准备模板 → 配置设置 → 完成

**技术实现**（对产品经理隐藏）：

```bash
prdkit init <target-dir> \
  --name "<projectName>" \
  --author "<author>" \
  --non-interactive
# 可选字段有值时追加：
# --description, --product-positioning, --team-size, --project-stage
```

注意：目录不存在时先 `mkdir -p`；已存在且不为空时询问用户；不要传空值或"暂不设置"的字段。

### 5. 启动预览服务器

显示「正在启动…→ 已启动，访问地址 http://localhost:<port>」

**技术实现**（对产品经理隐藏）：

```bash
cd <target-dir> && prdkit serve --no-open
```

注意：`prdkit serve` 自动在 7788-7888 范围查找可用端口，从输出或 `.prdkit/server.pid` 中捕获实际地址填入 `<port>` 再展示。

### 6. 引导登录云端

告知产品经理可以登录云端平台，在网页上管理项目、预览原型、邀请团队成员协作。询问是否需要打开登录页面，或以后再体验。

**技术实现**：如用户同意，使用 Bash 直接打开浏览器访问后端登录地址（如 `http://localhost:3001/login` 或生产环境地址）。

### 7. 快速开始指南

展示项目结构、预览服务器地址，告知产品经理后续可直接提出需求（写 PRD、做原型、登录云端等），AI 会调用对应 skill 处理。

## 参考：可用技能列表

AI 助手通过 Skill 工具调用以下 skill，不要让产品经理使用命令行。

| 操作 | Skill | 说明 |
|------|-------|------|
| 写 PRD | `prdkit-prd-create` | 创建产品需求文档 |
| 做原型 | `prdkit-page-create` | 生成页面原型 HTML |
| 修改原型 | `prdkit-page-update` | 修改已有原型 |
| 添加标注 | `prdkit-mark-create` | 在原型上添加标注 |
| 修改标注 | `prdkit-mark-update` | 修改原型上的标注说明 |

> 此列表会持续扩展。

## 注意事项

1. **语言**：用产品经理能理解的语言，避免技术术语（CLI、配置文件等）。用"文件夹""编辑器""设置"
2. **交互**：每问题给说明和示例，提供默认值，允许跳过，用友好措辞
3. **技术隐藏**：不展示命令行输出和错误信息，自动处理所有技术细节
4. **错误处理**：用友好语言解释问题，提供明确解决方案，能自动修复就自动修复
5. **成功引导**：完成后提供清晰的下一步指引，鼓励立即开始使用

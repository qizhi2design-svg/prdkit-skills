# prdkit-page-update Skill

## 概述

当用户通过 viewer 提供元素信息、页面路径或 HTML 片段，并要求修改现有原型页面时，使用此 skill 进行精准更新。

## 当前方法

1. 解析元素信息
2. 定位 `index.html`、相关脚本和示例数据
3. 通过结构化提问明确修改意图
4. 直接完成页面编辑
5. 需要时创建 prototype checkpoint
6. 用当前项目可用的方式完成预览验证

## 核心特性

### 1. 精准定位
- 基于元素选择器和文件路径快速定位代码位置
- 同时分析 HTML、交互逻辑和数据关联

### 2. 结构化提问
- 聚焦修改操作、数据变化、交互变化、视图变化
- 提供具体选项而非开放式问题

### 3. 快速执行
- 明确意图后直接修改，不额外制造冗长流程
- 按数据 → 逻辑 → 视图的顺序推进

### 4. 版本留痕
- 重要改动后创建 checkpoint
- checkpoint message 用产品语言概括业务变化

## 与 create-page skill 的区别

| 维度 | create-page | update-page |
|------|-------------|-------------|
| 触发时机 | 创建新页面 | 修改现有页面 |
| 输入信息 | 页面需求描述 | 元素信息 + 修改目标 |
| 工作重点 | 整体布局和结构 | 精准定位和修改 |
| 工作流程 | 方案稿 → 创建 → 补全 | 解析 → 澄清 → 直接修改 |

# 创建原型标注

## 工作流程

### 智能判断用户意图

首先分析用户输入，判断是否需要交互确认：

**直接创建标注的情况**（跳过确认，直接进入标注内容收集）：
- 用户明确提到"创建标注"、"加标注"、"标注一下"等关键词
- 用户提供了元素信息 + 功能描述/交互说明/数据规则
- 用户的描述中包含明确的需求内容（如"这个按钮点击后..."、"需要显示..."等）

**需要确认的情况**（进入分析和确认流程）：
- 用户只提供了元素信息，没有说明要做什么
- 发现该元素已有标注
- 用户的意图不明确（可能是要修改页面、修改标注或创建标注）

### 流程 A：直接创建标注（用户意图明确）

当用户意图明确时，直接收集标注信息并创建：

1. **解析元素信息**：
   - 提取 CSS 选择器（从 HTML 标签或用户提供的选择器）
   - 提取 DOM 路径（如果提供）
   - 确定原型路径（如果在 prdkit 项目中）

2. **分析用户描述，提取标注内容**：
   - 从用户输入中提取功能描述
   - 识别交互行为说明
   - 识别数据规则和异常处理
   - 生成合适的标注标题

3. **如果信息完整，直接创建标注**：
   - 构建标注标题（从用户描述中提取或生成）
   - 整理标注描述（结构化用户提供的内容）
   - 执行 `prdkit prototype mark create` 命令
   - 提示用户标注已创建

4. **如果信息不完整，补充提问**：
   - 只询问缺失的关键信息
   - 使用简短的提问，不要重复用户已提供的内容

### 流程 B：分析确认流程（用户意图不明确）

当需要确认用户意图时：

1. **解析用户提供的元素信息**：
   - 项目名称
   - 文件路径（原型目录的绝对路径）
   - DOM 路径（元素的 DOM 层级）
   - 元素选择器（可选）
   - 标记文件路径（如果已有标注）

2. **提取关键信息**：
   - 从文件路径中提取原型的相对路径
   - 例如：`/Users/xxx/workspace/prototypes/车后本地生活门店知识库` → `车后本地生活门店知识库`
   - 判断是否已有标注

3. **检查现有标注**：
   ```bash
   prdkit prototype mark list --prototype <原型路径> --json
   ```
   - 如果发现该元素已有标注，读取并展示

4. **简要分析并确认操作**：
   
   使用 `AskUserQuestion` 确认用户想做什么：
   - 创建新标注（记录业务需求、交互说明）
   - 修改现有标注（更新已有标注内容）
   - 修改页面代码（改变元素样式或交互）
   - 暂不操作

5. **根据用户选择决定下一步**：
   - 创建新标注 → 继续收集标注内容
   - 修改现有标注 → 提示使用 `/prdkit-mark-update`
   - 修改页面代码 → 提示使用 `/prdkit-page-update`
   - 暂不操作 → 结束流程

### 收集标注内容

无论通过哪个流程，最终都需要收集以下信息：

1. **构建 CSS 选择器**：
   - 从 HTML 标签中提取（如 `<button class="login-btn">` → `.login-btn`）
   - 或使用用户提供的选择器
   - 或从 DOM 路径转换

2. **确定标注标题**：
   - 优先从用户描述中提取关键词
   - 或根据元素类型和功能生成
   - 保持简洁（5-15 字）

3. **整理标注描述**：
   - 结构化用户提供的内容
   - 使用 Markdown 格式
   - 包含：功能说明、交互行为、数据规则、异常处理等

4. **如果信息不完整，简短提问**：
   - 只询问缺失的关键信息
   - 不要重复用户已提供的内容
   - 提供选项帮助用户快速选择

### 创建标注

1. **构建并执行命令**：
   ```bash
   prdkit prototype mark create \
     --prototype <原型路径> \
     --title "<标注标题>" \
     --selector "<CSS选择器>" \
     --desc "<标注描述>"
   ```
   
   或使用 Markdown 文件：
   ```bash
   prdkit prototype mark create \
     --prototype <原型路径> \
     --title "<标注标题>" \
     --selector "<CSS选择器>" \
     --desc-file <markdown文件路径>
   ```

2. **检查执行结果**：
   - 成功：提示标注已创建，告知文件位置
   - 失败：分析错误原因，提供解决建议

3. **友好提示**：
   - 标注文件位置
   - 可在 viewer 中查看
   - 已自动创建 checkpoint

## 标注类型

提供三种核心标注类型，详见 `references/templates/` 目录：

- **功能说明**：说明产品功能需求
- **交互行为说明**：描述用户交互流程和系统响应
- **数据展示说明**：说明数据来源、展示规则和异常处理

## 约束规则

- **优先直接创建**：当用户意图明确时，不要过度提问，直接创建标注
- **智能提取信息**：从用户输入中提取标注标题和描述，减少重复询问
- **简化交互**：只在真正需要澄清时才使用 `AskUserQuestion`
- **保持聚焦**：标注内容应与选中的元素直接相关
- **结构化输出**：使用 Markdown 格式组织标注描述
- **相对路径**：原型路径必须是相对路径（相对于 workspace/prototypes/）

## 决策树

```
用户输入
    │
    ├─ 包含明确的功能描述？
    │   ├─ 是 → 提取信息 → 直接创建标注
    │   └─ 否 ↓
    │
    ├─ 在 prdkit 项目中？
    │   ├─ 是 → 检查现有标注
    │   │   ├─ 已有标注 → 确认操作类型
    │   │   └─ 无标注 → 收集标注内容
    │   └─ 否 → 提取选择器 → 收集标注内容
    │
    └─ 收集标注内容
        ├─ 信息完整 → 创建标注
        └─ 信息不完整 → 简短提问 → 创建标注
```

## 最佳实践

1. **标题简洁明确**：用一句话说清楚标注的核心内容
2. **描述结构化**：使用 Markdown 格式，分段落、列表组织内容
3. **聚焦当前元素**：标注内容应该与选中的元素直接相关
4. **提供上下文**：说明为什么需要这个标注，背景是什么

## 示例

**用户输入**：
```
项目: 测试项目
文件路径: /Users/xxx/workspace/prototypes/车后本地生活门店知识库
DOM 路径: body > div.app-shell > aside.sidebar > div.brand
```

**Skill 提问**：
```
关于这个元素的标注：

1. 标注类型：
   - A) 功能说明
   - B) 交互行为说明
   - C) 数据展示说明
   - D) 其他

2. 标注标题（一句话概括）：

3. 标注描述（详细说明）：
```

**用户回答**：
- 类型：A) 功能说明
- 标题：品牌展示区
- 描述：展示门店品牌信息，包括 logo 和名称

**执行命令**：
```bash
prdkit prototype mark create \
  --prototype "车后本地生活门店知识库" \
  --title "品牌展示区" \
  --selector "body > div.app-shell > aside.sidebar > div.brand" \
  --desc "## 功能说明
展示门店品牌信息，帮助用户识别当前门店

## 展示内容
- 门店 logo
- 门店名称

## 交互行为
点击品牌区可返回首页"
```

## 注意事项

- 确保正确提取原型的相对路径
- 引导用户提供有价值的标注内容
- 告知用户标注创建后会自动创建版本快照

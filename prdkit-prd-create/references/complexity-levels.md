# PRD 复杂度分级

## L1：单点优化 / 局部变更

适用场景：
- 单个按钮、单条文案、单个表单步骤优化
- 已有流程中的局部补充

输出要求：
- 背景与问题
- 目标与成功标准
- In/Out Scope
- 功能需求
- 风险与验收

## L2：单模块完整需求

适用场景：
- 列表页、详情页、审批页、设置页等单模块需求
- 功能边界清晰，不需要大量跨团队协同

输出要求：
- L1 全部章节
- 用户/角色与场景
- 交互与内容要求
- 数据/埋点/指标

## L3：跨模块协同 / 中型项目

适用场景：
- 一个需求需要列表、详情、配置、通知等多个模块配合
- 涉及多个角色或多个状态流转

输出要求：
- L2 全部章节
- 业务规则 / 流程
- 外部依赖
- 风险拆解
- 验收标准完整化

## L4：平台级 / 多角色 / 复杂流程

适用场景：
- 权限系统、平台规则引擎、多角色工作流
- 跨团队、跨系统、跨阶段建设

输出要求：
- L3 全部章节
- 多角色视角
- 完整流程与异常流
- 阶段性里程碑
- 高复杂度自检

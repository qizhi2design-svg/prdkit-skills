# 高复杂度自检与待完善清单

适用于 L3 / L4，在正式 PRD 输出前执行。

## 1. 结构链路检查

- 背景与问题是否能支撑目标与成功标准
- 目标与成功标准是否能落到功能需求与验收标准
- 范围、依赖、权限、上线策略是否彼此一致

## 2. 重大风险快扫

- 是否缺少边界条件
- 是否缺少异常流
- 是否缺少外部依赖
- 是否缺少关键角色或权限说明
- 是否缺少数据口径、埋点或效果指标
- 是否缺少上线、回滚或灰度策略

## 3. 按产品类型补充

- 商业化产品：是否缺少客户价值、竞品/替代方案、增长或续费指标
- 企业自研系统：是否缺少 ROI、组织协同影响、内部推广方案
- 交易型 / 业务型：是否缺少流程图、状态流转、关键规则边界
- 基础服务型：是否缺少 API 边界、调用约束、稳定性和隔离方案

## 4. 输出要求

在正式 PRD 末尾补一个“附：待完善清单”，按优先级分三类：

- 🔴 必须补充：影响评审与落地的关键缺口
- 🟡 建议补充：影响方案质量但不阻塞评审
- 🟢 可选完善：锦上添花类补充

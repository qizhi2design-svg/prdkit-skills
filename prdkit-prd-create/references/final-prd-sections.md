# 正式 PRD 章节规则

## 通用基础章节

所有复杂度等级都至少覆盖：
- 背景与问题
- 目标与成功标准
- 范围
- 功能需求
- 风险与依赖
- 验收标准

## L1

建议保留：
- 背景与问题
- 目标与成功标准
- 范围
- 功能需求
- 验收标准
- 风险与依赖

可简化：
- 用户/角色与场景
- 数据/埋点/指标
- 里程碑

## L2

建议保留：
- 基础章节全部
- 用户/角色与场景
- 交互与内容要求
- 数据/埋点/指标

## L3

必须增加：
- 业务规则 / 流程
- 外部依赖说明
- 详细风险
- 更完整的验收标准

## L4

必须增加：
- 多角色流程
- 异常流 / 边界情况
- 分阶段里程碑
- 高复杂度自检

## 高复杂度自检清单

适用于 L3 / L4：
- 是否说明了边界条件
- 是否说明了外部依赖
- 是否说明了异常流
- 是否给出了可执行验收标准

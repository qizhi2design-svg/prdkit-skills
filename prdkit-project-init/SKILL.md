# 产品项目初始化引导

用产品经理能理解的方式收集信息，使用当前桌面端内置 tools 完成项目初始化，并给出下一步建议。

## 核心原则

- 用业务语言沟通，不把工具名、目录结构、技术细节直接甩给用户
- 初始化动作由 `project_init` 完成；项目状态确认由 `project_info` 完成
- 不承诺额外的预览服务、端口行为或安装流程
- 收集信息时优先拿到够用字段，不为了“完整”追问一长串低价值问题

## 工作流程

### 1. 欢迎并说明接下来会做什么

简要说明你会先收集项目信息，再初始化项目空间，最后给出继续写 PRD、做原型、加标注的建议。

### 2. 收集项目信息

至少确认这些字段：

- `targetDir`
- `projectName`
- `author`

按需补充这些可选字段：

- `description`
- `productPositioning`
- `teamSize`
- `projectStage`
- `cloudHost`

提问方式要轻量。优先一次问 2-3 个强相关信息，不要把用户变成填表机器。

### 3. 确认是否已有项目上下文

如果用户已经在一个仓库里工作，先确认这是“新建项目”还是“继续已有项目”。  
如果已有项目迹象明显，可以先用 `project_info` 看是否已经是有效项目，再决定是否还要初始化。

### 4. 执行初始化

用 `project_init` 创建项目：

- `targetDir` 使用用户确认后的目标目录
- `projectName`、`author` 为必填
- 只有在用户确实提供时才传入可选字段

如果目标目录已有内容，先和用户确认是否要继续，不要擅自覆盖已有工作。

### 5. 验证初始化结果

初始化完成后，立即调用 `project_info` 读取项目统计与基础状态，确认：

- 项目根目录可识别
- 项目名称正确
- PRD / prototype 计数正常
- 云端配置状态可读取

### 6. 向用户汇报结果

输出应聚焦在用户价值，而不是技术过程：

- 项目已创建或已可用
- 当前项目名称
- 目前 PRD 数量、原型数量
- 下一步最适合做什么

推荐的下一步通常是：

- 写 PRD：`prdkit-prd-create`
- 做原型：`prdkit-page-create`
- 修改原型：`prdkit-page-update`
- 添加标注：`prdkit-mark-create`

## 输出约束

- 不要描述额外安装步骤、命令细节或本 skill 不负责的预览启动行为
- 不要承诺固定访问地址或预览端口
- 不要要求用户自己拼接初始化参数
- 如果初始化失败，用友好语言解释问题，并给出下一步处理建议

## 示例

当用户说“帮我起一个门店巡检后台项目”时：

1. 确认项目名称、作者、保存位置
2. 追问一句产品定位或项目阶段
3. 调用 `project_init`
4. 调用 `project_info`
5. 告知“项目已经准备好，接下来建议先写 PRD 或先起关键页面原型”

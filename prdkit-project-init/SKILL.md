# 产品项目初始化引导

为产品经理提供友好的项目初始化体验，用简单的问答方式收集信息，自动完成所有技术配置，让产品经理可以立即开始工作。

## 使用场景

- 产品经理要开始一个新产品项目
- 第一次使用 prdkit 工具
- 需要为产品文档创建一个管理空间

## 设计原则

1. **语言友好**：避免技术术语，用产品经理熟悉的语言
2. **步骤清晰**：每一步都有明确的说明和示例
3. **自动化**：技术细节全部自动处理，产品经理只需回答简单问题
4. **即时可用**：完成后立即可以开始工作，无需额外配置

## 执行步骤

### 1. 欢迎与说明

用友好的语言向产品经理说明：

```
👋 欢迎使用 prdkit！

prdkit 是一个专为产品经理设计的文档管理工具，可以帮你：
✅ 管理 PRD（产品需求文档）
✅ 制作和预览原型
✅ 记录讨论和 Bug
✅ 追踪版本变化

接下来我会问你几个简单的问题，然后自动帮你创建一个项目空间。
整个过程大约 2 分钟，之后你就可以开始工作了！
```

### 2. 收集项目信息

用简单的问答方式收集信息，每个问题都提供清晰的说明和示例。

#### 必填信息

**问题 1：项目名称**
```
📝 请给你的项目起个名字

这个名字会显示在所有文档的顶部，帮助团队成员快速识别项目。

示例：
  - 电商平台
  - 企业管理系统
  - 移动 App
  - 客户服务系统
```

**问题 2：你的名字**
```
👤 请输入你的名字或团队名称

这会显示为文档的创建者，方便团队协作时知道是谁负责。

示例：
  - 张三
  - 产品团队
  - Alice
```

#### 可选信息（可以跳过，以后再补充）

**问题 3：项目简介**
```
📄 用一两句话描述这个项目（可选）

这个描述会帮助新加入的团队成员快速了解项目是做什么的。

示例：
  - 面向中小企业的电商解决方案
  - 帮助销售团队管理客户的 CRM 系统
  - 为用户提供在线学习的教育平台

💡 提示：如果现在不确定，可以跳过，以后随时可以补充
```

**问题 4：产品类型**
```
🎯 请选择最符合你产品的类型（可选）

这有助于我们为你推荐合适的文档模板和最佳实践。

选项：
  ○ 企业内部系统 - 业务管理类（如 ERP、CRM、OA）
  ○ 企业内部系统 - 工具效率类（如协作工具、数据分析）
  ○ 企业内部系统 - 数据分析类（如 BI、报表系统）
  ○ 面向用户的产品 - 内容社区（如论坛、社交平台）
  ○ 面向用户的产品 - 电商交易（如购物平台、支付系统）
  ○ 面向用户的产品 - 工具应用（如效率工具、生活服务）
  ○ 暂时不确定，以后再选

💡 提示：选择后可以随时修改
```

**问题 5：团队规模**
```
👥 你的团队有多少人？（可选）

这有助于我们推荐合适的协作方式。

选项：
  ○ 就我一个人
  ○ 2-5 人的小团队
  ○ 6-15 人的中型团队
  ○ 15 人以上的大团队
  ○ 暂时不确定

💡 提示：这不会影响工具的使用，只是帮助我们提供更好的建议
```

**问题 6：项目阶段**
```
🚀 项目目前处于什么阶段？（可选）

选项：
  ○ 规划阶段（刚开始，还在想怎么做）
  ○ 开发阶段（已经开始做了，正在实现功能）
  ○ 迭代优化（已经上线，在持续改进）
  ○ 维护阶段（功能稳定，主要是修 Bug）
  ○ 暂时不确定

💡 提示：这只是帮助你更好地组织文档，可以随时调整
```

**问题 7：项目位置**
```
📁 项目文件要保存在哪里？

建议：
  - 如果不确定，直接按回车使用当前位置
  - 如果想指定位置，输入完整路径（如：/Users/你的名字/Documents/我的项目）

默认：当前位置

💡 提示：项目会创建一个文件夹，里面包含所有文档和配置
```


使用 `AskUserQuestion` 询问：
- 问题：`确认信息并开始创建项目？`
- 选项：
  - `确认，开始创建`
  - `我想修改一下`

### 4. 执行初始化（自动完成，显示进度）

向产品经理展示友好的进度提示：

```
🚀 正在创建项目...

[1/4] 📁 创建项目文件夹...
[2/4] 📝 准备文档模板...
[3/4] ⚙️  配置项目设置...
[4/4] ✅ 完成！
```

**技术实现**（对产品经理隐藏）：

使用 Bash 工具调用 `prdkit init` 命令，只传递用户实际填写的参数：

```bash
# 基础命令
prdkit init <target-dir> \
  --name "<projectName>" \
  --author "<author>" \
  --non-interactive

# 如果用户填写了可选字段，则添加：
# --description "<description>" \
# --product-positioning "<productPositioning>" \
# --team-size "<teamSize>" \
# --project-stage "<projectStage>"
```

**注意**：
- 如果目标目录不存在，先使用 `mkdir -p` 创建
- 如果目录已存在且不为空，友好地询问用户是否继续
- 不要为空值或"暂不设置"的字段传递参数
- 所有技术细节和命令输出不要展示给产品经理

### 5. 启动预览服务器（自动完成）

向产品经理展示友好的提示：

```
🌐 正在启动预览服务器...

✅ 预览服务器已启动！
   访问地址：http://localhost:3000

💡 提示：
   - 预览服务器会自动监听文件变化
   - 你修改原型后，浏览器会自动刷新
   - 关闭终端前，服务器会一直运行
```

**技术实现**（对产品经理隐藏）：

```bash
cd <target-dir> && prdkit serve --no-open
```

**注意**：
- 使用 `--no-open` 避免自动打开浏览器（稍后会引导用户访问）
- 如果端口被占用，自动尝试其他端口（3001, 3002...）
- 记录实际使用的端口号
- 不要展示命令行输出，只显示友好的提示信息

## 错误处理（用友好的语言）

### 目录已存在且不为空

```
⚠️  注意

这个位置已经有文件了：

如果继续，可能会覆盖这些文件。

建议：
  1. 换一个新位置（推荐）
  2. 备份现有文件后继续
  3. 确认这些文件可以被覆盖

```

### 网络连接问题

```
❌ 无法下载项目模板

可能的原因：
  - 网络连接不稳定
  - 防火墙阻止了连接

解决方法：
  1. 检查网络连接
  2. 稍后重试
  3. 如果问题持续，请联系技术支持

```

### 服务器启动失败

```
⚠️  预览服务器启动遇到问题

可能的原因：
  - 端口 3000 被其他程序占用了

解决方法：
  我会自动尝试其他端口（3001, 3002...）

正在重试...
```

### 编辑器打开失败

```
⚠️  无法自动打开编辑器

可能的原因：
  - 编辑器还没有安装
  - 编辑器命令没有配置好

没关系，你可以手动打开：

📁 项目位置：<absolute-path>

在编辑器中选择"打开文件夹"，然后选择上面的路径就可以了。

💡 如果你还没有编辑器，推荐：
   - VS Code（免费）：https://code.visualstudio.com/
   - Cursor（免费，有 AI）：https://cursor.sh/
```

## 注意事项（给 AI 助手的指导）

1. **语言风格**：
   - 始终使用产品经理能理解的语言
   - 避免技术术语（如：CLI、配置文件、命令行参数等）
   - 用"文件夹"而不是"目录"
   - 用"编辑器"而不是"IDE"
   - 用"设置"而不是"配置"

2. **交互体验**：
   - 每个问题都提供清晰的说明和示例
   - 提供合理的默认值，让产品经理可以快速跳过
   - 允许跳过所有可选项
   - 用友好的 emoji 增强可读性

3. **技术细节隐藏**：
   - 不要展示命令行输出
   - 不要展示技术错误信息
   - 将技术问题转换为产品经理能理解的语言
   - 自动处理所有技术配置

4. **进度反馈**：
   - 每个步骤都提供清晰的进度提示
   - 使用进度条或步骤编号
   - 告诉产品经理正在做什么，为什么要等待

5. **错误处理**：
   - 用友好的语言解释问题
   - 提供明确的解决方案
   - 如果可以自动修复，就自动修复
   - 避免让产品经理感到困惑或挫败

6. **成功引导**：
   - 完成后提供清晰的下一步指引
   - 用具体的例子而不是抽象的说明
   - 鼓励产品经理立即开始使用
   - 提供快速参考指南

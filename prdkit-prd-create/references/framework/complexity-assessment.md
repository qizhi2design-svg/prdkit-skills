# PRD 复杂度评估与章节适配

## 1. 先用用户视角描述变更

在判断复杂度前，先用一句话概括用户感知到的变化：

`{角色} 在 {场景} 下，可以 {做什么 / 感知到什么变化}`

优先看影响面，不优先看实现细节。

## 2. L1-L4 决策树

按影响面逐级判断，命中即停：

1. 是否新建系统 / 新业务域 / 完整子系统？
   - 是：L4
2. 是否新增页面、新增核心实体、新增角色/权限、重构流程结构？
   - 是：L3
3. 是否修改或新增业务规则，但不引入新页面和新核心实体？
   - 是：L2
4. 是否只是配置、文案、阈值、样式、轻交互调整？
   - 是：L1

## 3. 边界消歧

- 新增视图但没有新增业务实体或角色：更偏 L2
- 一条业务规则影响多个页面，但本质还是同一规则：通常仍是 L2
- 流程结构变化且角色分工变化：通常升到 L3
- 新增完整模块但无独立业务闭环：通常 L3
- 新增完整业务闭环或子系统：通常 L4

## 4. 辅助信号

当决策树结论模糊时，用这些信号辅助：

| 维度 | L1 | L2 | L3 | L4 |
| --- | --- | --- | --- | --- |
| 新增页面数 | 0 | 0 | 1-3 | 4+ |
| 新增核心实体数 | 0 | 0 | 1-3 | 4+ |
| 新增/变化角色数 | 0 | 0 | 0-1 | 2+ |
| 跨系统集成 | 无 | 无 | 0-1 | 2+ |
| 流程变更范围 | 无 | 单节点规则 | 局部流程 | 端到端重构 |

## 5. 级别定义

### L1：配置级

- 单点优化 / 局部变更
- 常见于文案、阈值、样式、轻交互

### L2：规则级

- 修改或新增规则
- 不引入新页面 / 新核心实体

### L3：模块级

- 新增页面 / 模块
- 新增核心实体 / 角色权限
- 涉及流程结构调整

### L4：系统级

- 新建业务域 / 子系统
- 平台级、多角色、复杂流程

## 6. 章节适配总表

| 章节能力 | L1 | L2 | L3 | L4 |
| --- | --- | --- | --- | --- |
| 背景与问题 | 必选 | 必选 | 必选 | 必选 |
| 目标与成功标准 | 必选 | 必选 | 必选 | 必选 |
| 用户/角色与场景 | 精简 | 必选 | 必选 | 必选 |
| 范围（In/Out） | 必选 | 必选 | 必选 | 必选 |
| 功能需求 | 必选 | 必选 | 必选 | 必选 |
| 业务规则 / 流程 | 按需 | 精简 | 必选 | 必选 |
| 数据 / 埋点 / 指标 | 按需 | 必选 | 必选 | 必选 |
| 权限 / 角色设计 | 免除 | 按需 | 精简 | 必选 |
| 风险与依赖 | 精简 | 必选 | 必选 | 必选 |
| 里程碑 / 上线与回滚 | 免除 | 按需 | 精简 | 必选 |
| 待完善清单 / 自检 | 免除 | 免除 | 必选 | 必选 |

## 7. 产品类型叠加规则

复杂度等级确定后，再叠加产品定型：

- 商业化产品：增加市场、客户、商业目标、增长与续费视角
- 企业自研系统：增加 ROI、效率提升、内部推广和采纳视角
- 业务型 / 交易型：强调流程、状态机、角色权限、核心数据模型
- 工具型：强调效率和低学习成本
- 基础服务型：强调 API、稳定性、扩展性、隔离与监控

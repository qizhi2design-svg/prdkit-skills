# 交互行为标注模板

用于描述用户交互流程和系统响应。

## 基础模板

```markdown
## 交互说明
[描述交互的触发条件和响应]

## 状态变化
[描述交互前后的状态变化]

## 边界情况
[描述特殊情况下的交互行为]
```

## 详细模板

```markdown
## 交互说明
**触发条件**: [什么操作触发]
**交互响应**: [触发后发生什么]
**反馈形式**: [视觉/听觉/触觉反馈]

## 状态变化
### 初始状态
[描述初始状态]

### 交互中状态
[描述交互过程中的状态]

### 完成状态
[描述交互完成后的状态]

## 边界情况
- **情况1**: [描述] → [处理方式]
- **情况2**: [描述] → [处理方式]

## 可访问性
- 键盘操作: [如何用键盘完成交互]
- 屏幕阅读器: [如何朗读]
```

## 示例：下拉菜单交互

```markdown
## 交互说明
**触发条件**: 点击下拉框或按下空格键
**交互响应**: 展开选项列表，高亮当前选中项
**反馈形式**: 视觉（展开动画、高亮）

## 状态变化
### 初始状态
- 下拉框显示当前选中值或占位文本
- 选项列表隐藏

### 交互中状态
- 选项列表展开
- 鼠标悬停或键盘导航时高亮选项
- 点击选项或按回车键选中

### 完成状态
- 下拉框显示新选中的值
- 选项列表收起
- 触发 onChange 事件

## 边界情况
- **选项为空**: 显示「暂无选项」
- **选项过多**: 支持滚动，显示滚动条
- **搜索无结果**: 显示「未找到匹配项」
- **禁用状态**: 不响应点击，显示禁用样式

## 可访问性
- 键盘操作: 空格/回车展开，上下键导航，回车选中，Esc关闭
- 屏幕阅读器: 朗读「下拉框，当前选中：XXX」
```

## 使用建议

1. **明确触发条件**：清楚说明什么操作会触发交互
2. **描述状态变化**：说明交互前、中、后的状态
3. **考虑边界情况**：列出特殊情况下的处理方式
4. **关注可访问性**：说明键盘操作和屏幕阅读器支持
5. **反馈及时性**：说明用户能得到什么反馈

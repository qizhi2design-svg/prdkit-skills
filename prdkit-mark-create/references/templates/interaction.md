# 交互行为标注模板

用于描述用户交互流程和系统响应。

## 基础结构

```markdown
## 交互说明
[描述交互的触发条件和响应]

## 状态变化
[描述交互前后的状态变化]

## 边界情况
[描述特殊情况下的交互行为]
```

## 详细结构

```markdown
## 交互说明
**触发条件**: [什么操作触发]
**交互响应**: [触发后发生什么]
**反馈形式**: [视觉/听觉/触觉反馈]

## 状态变化
### 初始状态
[描述初始状态]

### 交互中状态
[描述交互过程中的状态]

### 完成状态
[描述交互完成后的状态]

## 边界情况
- **情况1**: [描述] → [处理方式]
- **情况2**: [描述] → [处理方式]
```

## 使用建议

1. 明确触发条件：清楚说明什么操作会触发交互
2. 描述状态变化：说明交互前、中、后的状态
3. 考虑边界情况：列出特殊情况下的处理方式
4. 反馈及时性：说明用户能得到什么反馈

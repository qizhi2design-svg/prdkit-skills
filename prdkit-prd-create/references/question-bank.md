# 需求澄清题库

总共提 3-4 个问题即可，问题必须具体、可回答、能直接影响复杂度判断或章节裁剪。

## 第一优先确认

- 这是单点优化、单模块需求，还是跨模块 / 平台级方案？

## 背景与目标

- 当前要解决的核心业务问题是什么？
- 如果这次不做，会造成什么实际影响？
- 这个需求最关键的结果指标是什么？

## 用户与角色

- 谁会直接使用这份能力？是否有多个角色参与？
- 不同角色看到的内容或可执行动作是否不同？

## 流程与范围

- 这是一个独立页面/模块，还是会串联多个流程节点？
- 是否涉及跨页面、跨模块、跨系统协同？
- 哪些内容这次明确不做？

## 数据与依赖

- 关键数据从哪里来，要写回哪里？
- 是否依赖外部接口、权限系统、消息通知或审批流？

## 风险与验收

- 这类需求最怕出什么问题？
- 验收时最少要满足哪 3 条标准？

# 修订记录模板

## 放置位置

优先放在 PRD 标题区之后、正文之前。

如果文档里已有 `## 修订记录`，就在其末尾追加；否则创建。

## 推荐格式

```md
## 修订记录

| 修订日期 | 修订人 | 修订明细 |
| --- | --- | --- |
| 2026.4.29 | 黄启智 | 盘点汽车一门店一渠道组织现状，提供数据追踪方案 |
| 2026.3.22 | 黄启智 | 试用转正方案整体设计 |
```

## 追加规则

如果 PRD 已有 `## 修订记录` 表格：

- 保留原有表头
- 在表格末尾追加一行
- 不要改写已有历史记录

如果 PRD 已有 `## 修订记录` 但不是表格：

- 优先将其整理成三列表格
- 列名固定为：`修订日期`、`修订人`、`修订明细`

如果 PRD 没有 `## 修订记录`：

- 新建该章节
- 创建三列表格
- 写入本次修订一行

## 字段要求

### 修订日期

- 使用 `YYYY.M.D` 或与现有文档一致的日期格式
- 若文档已有历史格式，优先沿用已有格式

### 修订人

- 优先使用 PRD frontmatter 中的 `creator`
- 没有时使用本次执行者名字
- 再没有时可回退为 `AI 协作更新`

### 修订明细

用一句或两句完整话描述，不拆成子 bullet。

应尽量包含：

- 来源 prototype 或页面名称
- 本次核心改动
- 是否同步了标注内容
- 对应功能描述是新增、细化还是替换

可参考写法：

- `根据 prototype「门店知识库」，同步补充筛选区与任务列表的功能说明，细化操作流程与约束条件。`
- `根据 prototype「支付结果页」更新结果回执模块描述，补充状态反馈与短信通知待确认事项。`

## 语言要求

- 修订记录要面向 PRD 阅读者，而不是面向实现者
- 避免只写“同步了一下原型”
- 优先写“改了什么”，其次写“来源于哪个 prototype / mark”
- 表格单元格中的文字要简洁，但要能独立表达清楚

# 功能说明标注模板

用于描述产品功能需求和规格说明。

## 基础模板

```markdown
## 功能说明
[功能的核心价值和目标用户]

## 交互流程
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 注意事项
- [注意点1]
- [注意点2]
```

## 详细模板

```markdown
## 功能说明
**目标用户**: [谁会使用这个功能]
**核心价值**: [这个功能解决什么问题]
**使用场景**: [在什么情况下使用]

## 交互流程
### 正常流程
1. [步骤1]
2. [步骤2]
3. [步骤3]

### 异常流程
- **场景1**: [异常情况] → [处理方式]
- **场景2**: [异常情况] → [处理方式]

## 数据说明
- **输入**: [需要什么数据]
- **输出**: [产生什么数据]
- **数据来源**: [数据从哪里来]

## 注意事项
- [注意点1]
- [注意点2]
- [边界情况说明]

## 相关资源
- PRD: [链接]
- 设计稿: [链接]
- 讨论: [链接]
```

## 示例：用户登录功能

```markdown
## 功能说明
**目标用户**: 所有需要访问系统的用户
**核心价值**: 验证用户身份，保护系统安全
**使用场景**: 用户首次访问或会话过期时

## 交互流程
### 正常流程
1. 用户输入手机号和密码
2. 点击「登录」按钮
3. 系统验证身份
4. 验证成功后跳转到首页

### 异常流程
- **手机号格式错误**: 输入框下方显示「请输入正确的手机号」
- **密码错误**: 显示「手机号或密码错误，请重试」
- **账号被锁定**: 显示「账号已被锁定，请联系客服」

## 数据说明
- **输入**: 手机号（11位）、密码（6-20位）
- **输出**: 用户 token、用户基本信息
- **数据来源**: 用户输入 + 后端验证

## 注意事项
- 密码输入框需要隐藏显示
- 登录按钮在输入为空时禁用
- Token 需要安全存储
- 连续输错3次后显示验证码
- 连续输错5次后锁定账号30分钟
- 网络异常时有友好提示

## 相关资源
- PRD: [链接到登录功能 PRD]
- 设计稿: [链接到登录页设计稿]
```

## 使用建议

1. **明确目标用户**：清楚说明谁会使用这个功能
2. **突出核心价值**：解释功能解决什么问题
3. **完整的流程**：包含正常流程和异常流程
4. **注意边界情况**：在注意事项中说明特殊情况和边界条件
5. **关联资源**：链接到相关的 PRD、设计稿等文档
